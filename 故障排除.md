# 🆘 遇到问题？解决方案

## 问题 1：双击脚本没反应

### Windows 解决方案：

**方法 1：右键管理员运行**
- 右键点击 `auto-deploy.bat`
- 选择"以管理员身份运行"

**方法 2：用命令行运行**
- 按 `Win + R`
- 输入 `cmd` 回车
- 进入项目文件夹：
  ```cmd
  cd 你项目的路径
  ```
- 运行脚本：
  ```cmd
  auto-deploy.bat
  ```

**方法 3：用 PowerShell 运行**
- 右键点击文件夹空白处
- 选择"在终端中打开"
- 运行：
  ```powershell
  .\auto-deploy.bat
  ```

### Mac/Linux 解决方案：

```bash
# 给脚本添加执行权限
chmod +x auto-deploy.sh

# 运行脚本
./auto-deploy.sh
```

## 问题 2：提示找不到 Node.js

### 解决方案：

1. 访问：https://nodejs.org/
2. 点击 LTS 版本下载
3. 安装（一路 Next）
4. 重新打开终端/命令行
5. 再次运行脚本

## 问题 3：提示找不到 pnpm

### 解决方案：

**方法 1：自动安装**
脚本会自动安装，等待完成即可

**方法 2：手动安装**
```bash
npm install -g pnpm
```

## 问题 4：登录 Vercel 失败

### 解决方案：

**方法 1：用 GitHub 登录（最简单）**
1. 访问：https://vercel.com
2. 点击 "Sign Up"
3. 选择 "Continue with GitHub"
4. 授权登录
5. 回到命令行重新运行脚本

**方法 2：手动登录后再部署**
```bash
# 先单独登录
vercel login

# 登录成功后再部署
vercel
```

---

## 问题 5：部署时出错

### 常见错误 1：端口被占用
```bash
# 用 --yes 参数跳过确认
vercel --yes
```

### 常见错误 2：依赖安装失败
```bash
# 清理缓存重新安装
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

### 常见错误 3：构建失败
```bash
# 手动构建测试
pnpm build
```

## 问题 6：脚本一直卡住

### 解决方案：

按 `Ctrl + C` 停止，然后重新运行脚本

## 问题 7：部署成功但网站打不开

### 解决方案：

1. 检查网址是否正确
2. 等待 1-2 分钟（DNS 解析需要时间）
3. 清除浏览器缓存后重试
4. 查看控制台是否有错误信息

## 🎯 还是搞不定？

别着急！按照这个顺序：

### 第 1 步：检查环境
```bash
node --version    # 应该显示 v18.0 或更高
pnpm --version    # 应该显示 9.0 或更高
```

### 第 2 步：手动运行命令
```bash
pnpm install
npm install -g vercel
vercel login
vercel
```

### 第 3 步：查看详细日志
```bash
# Windows
auto-deploy.bat 2>&1 > log.txt

# Mac/Linux
./auto-deploy.sh 2>&1 > log.txt

# 把 log.txt 发给我分析
```
